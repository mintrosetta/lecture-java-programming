Java IO (input/output)
    โดยปกติ Java Program จะทำกายถ่ายโอนข้อมูลระหว่าง input/output จากอุปกรณ์หรือทรัพยากรณ์บางอย่าง
    เช่น keyboard, memory, monitor, file

What are stream ?
    เป็น Flow of data หรือ การไหลของข้อมูล
    โดยอาจะเป็นการไหลของข้อมูลจากทรัพยากรไปยังโปรแกรม
    หรือเป็นการไหลจากโปรแกรมไปให้กับทรัพยากร

    เช่น user ใส่ input ผ่านทาง keyboard โดยสิ่งที่ผู้ใช้ใส่มาจะถูกนำไปแสดงผลบน monitor
    user -> [type keyboard] -> [program] -> [monitor]

Buffer
    ในการส่งข้อมูลที่เครื่องผู้ส่งกับผู้รับมีระยะทางที่ไกลกัน จะเกิดปัญหาเรื่องของความเร็วของการส่งข้อมูล
    โดยที่ความเร็วของผู้ส่งและผู้รับอาจแตกต่างกันไป ซึ่งทำให้เกิดปัญหาความล่าชาของข้อมูลและอาจะทำให้
    ข้อมูลสูญหายละหว่างนำส่ง

    ตัวอย่าง
        ผู้ส่ง: ช้า
        ผู้รับ: เร็ว
        ผลลัพธ์: ผู้รับจะต้องระบข้อมูล ทำให้เกิดความล่าช้าในการประมวลผล
        --
        ผู้ส่ง: เร็ว 
        ผูรับ: ช้า
        ผลลัพธ์: เนื่องจากผู้รับ รับข้อมูลไม่ทัน ทำให้อาจเกิดการสูญหายระหว่างการนำส่งข้อมูล

    แนวทางการแก้ปํญหา
        เราจะใส่ข้อมูลที่เราต้องการส่งเข้าไปเก็บไว้ในกล่อง ๆ หนึ่งที่ชื่อว่า buffer ซึ่งคือ object หนึ่ง 
        ภายในหน่วยความจำเพื่อเก็บข้อมูลไว้ในระยะหนึ่ง โดยให้ให้โปรแกรมหรืออุปกรณ์อื่น ๆ เข้ามาดูข้อมูล
        ใน buffer ที่อยู่ใน memory แทน ทำให้ความเร็วของผู้ส่งและผู้รับแม้จะแต่งต่างกันมากเท่าไหร่ก็ไม่มีผล

    ** Buffer มีความสำคัญมากในการ stream data

    ตัวอย่าง
        เราสั่ง Java ให้เขียนไฟล์ลงไปใน Hdd 
        ตัว Java จะไม่ได้เขียนไฟล์ไปที่ Hdd ตรง ๆ แต่จะนำข้อมูลของไฟล์ใส่ไว้ที่ buffer ก่อน 
        แล้วข้อมูลของไฟล์ที่อยู่ใน buffer จะค่อย ๆ ไหลไปถูกนำไปสร้างไว้ใน Hdd โดยความเร็วในการไหล
        จะขึ้นอยู่กับ Hdd เขียนข้อมูลได้เร็วเท่าไหร่

How data flow in and out ?
    ข้อมูลจะถูกจัดส่งแบบทีละ byte จนกว่าข้อมูลจะครบ

Java Classes
    ByteStream: stream ข้อมูลในรูปแบบ byte
    InputStream/Reader: ข้อมูลที่จะถูก stream เข้ามาใน Java Program
    OutputStream/Writer: ข้อมูลที่จะส่งจาก Java Program ออกไป
    CharacterStream: stream ข้อมูลในรูปแบบ char (2 byte)
