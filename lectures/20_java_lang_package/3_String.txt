String
    จัดเก็บกลุ่มของตัวอักษร Collection of character
    โดยข้อมูลจะจัดเก็บเป็นแบบ immutable (ไม่สามารถเปลี่ยนแปลงได้)
    เมื่อมีการเปลี่ยนแปลงข้อมูลใหม่ จะเป็นการสร้าง object ใหม่ขึ้นมา

    โดยสาเหตุที่เป็น immutable เพราะทำให้ระบบสามารถประมวลผลได้เร็ว
    เนื่องจากไม่จำเป็นต้องทำการแก้ไขข้อมูลใน string เก่า 
    สามารถชี้ไปหา string ในตำแหน่งใหม่ได้เลย

    String a = "Hello";
    a = "World";

    จากโค้ดด้านบน Hello จะไม่ได้หายไปจาก memory 
    แต่ String reference ที่ขือว่า a ได้ชี้ไปหา String object ตัวใหม่ที่เก็บค่า World

StringBuffer
    เป็น mutable
    จะใช้ในกรณีที่ string ไม่มีความแน่นอนว่าจะเก็บข้อมูลกี่ตัวอักษร หรือจะมีการแก้ไข string อยู่เรื่อย ๆ

    โดย StringBuffer จะมี method ทั้งหมดของ string และมี method ที่ถูกเพิ่มมาอีกเพื่อจัดการกับ buffer
    เช่น 
        - append เพื่อทำการต่อ string 
        - insert เพื่อแทรก string เข้าไปใน index ที่ที่ระบุ

    แต่ว่าจะมีการระบุจำนวนความยอวของ string (capacity) และจำนวนตัวอักษรจะต้องไม่เกินจำนวน capacity

    สิ่งที่สำคัญของ StringBuffer คือ Thread Safe
        ข้อมูลสามารถเข้าถึงได้แค่ 1 thread เท่านั้น ทำให้ลดโอกาสที่ข้อมูลจะถูกเปลี่ยนแปลงหรือสูญหายเพราะ thread อื่น ๆ
        ถ้า thread อื่น ๆ ต้องการใช้ ต้องรอ thread ที่กำลังทำงาน ใช้งานให้เสร็จก่อน เช่น append ให้เสร็จก่อน จึงจะเข้ามาทำงานต่อได้
        แต่จะทำให้โปรแกรมช้าเล็กน้อย

StringBuilder
    เหมือนกับ StringBuffer แต่จะไม่ใช่รูปแบบของ Thread Safe
    ทำให้เร็วกว่า StringBuffer
